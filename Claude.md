# プロジェクト概要：「完全自動化されたマイ環境構築システム」
LANケーブルを挿して空のPC（またはVM）を起動するだけで、ネットワーク経由でArch Linuxがインストールされ、日常的に使用しているデスクトップ環境（Hyprland）や開発環境（Rust, Go, Zedエディタなど）が全自動でセットアップされる基盤を構築する。

# 使用する想定技術スタック
- 仮想化環境: VirtualBox または Proxmox (検証用)
- ネットワークプロトコル: PXE, DHCP, TFTP, HTTP
- サーバー側ミドルウェア: dnsmasq, Nginx (または Apache)
- ブートローダー: iPXE または Syslinux
- OS: Arch Linux
- プロビジョニング/IaC: シェルスクリプト または Ansible

# プロジェクトのフェーズ
- **Phase 1: 検証環境の準備**
  - プロビジョニングサーバー用VMと、空のクライアント用VMのネットワーク構築（内部ネットワークの設定など）。
- **Phase 2: ネットワークブート基盤の構築**
  - サーバー用VMへの `dnsmasq` の導入と DHCP/TFTP サーバーの設定。
  - クライアントVMをPXEブートさせ、Arch LinuxのLiveイメージ（archiso）をネットワーク越しに起動させる。
- **Phase 3: OSインストールの完全自動化**
  - 自動インストールスクリプトの作成。
  - Nginx等のHTTPサーバーを経由したスクリプトの配信と、起動時（カーネルパラメータ）の自動実行設定。
- **Phase 4: IaCによる環境構成管理**
  - OSインストール再起動後、Hyprland、Rust、Go、Zedエディタ、firefox、ドットファイルなどを自動で流し込むAnsibleプレイブック（またはシェルスクリプト）の作成。
- **Phase 5: ポートフォリオ用のドキュメント化**
  - GitHubのREADMEに記載するアーキテクチャ構成図の作成と、面接で語るべき技術的ポイントの言語化。

それでは、Phase 1の「検証環境の準備」から具体的な指示と解説をお願いします。
